# DPVO Encoder ONNX Models — Optimised

This folder contains losslessly optimised versions of the DPVO feature encoder ONNX models.
They are drop-in replacements for the originals in `../` — identical inputs, identical outputs, identical weights (no quantisation/pruning/... tomfoolery).

---

## Files

| File | Size | Description |
|------|------|-------------|
| `fnet.onnx` | ~723 KB | Feature encoder — optimized (was 727 KB) |
| `inet.onnx` | ~783 KB | Context encoder — optimized |


### `fnet.onnx` — Feature Encoder

- **Architecture:** `BasicEncoder4` — lightweight ResNet-style CNN, stride 4 total
- **Input:** `images` — `[batch, frames, 3, H, W]`, float32, normalized as `2*(x/255) - 0.5`
- **Output:** `fmap` — `[batch, frames, 128, H/4, W/4]`, float32
- **Purpose:** Produces 128-channel feature maps used for patch matching across frames.
  The `altcorr` CUDA kernel compares 3×3 patch neighborhoods from this map to estimate
  optical flow between keyframes.
- **Normalization:** InstanceNorm2d (computes per-batch stats — correct for odometry)

### `inet.onnx` — Context/Interest Encoder

- **Architecture:** Same `BasicEncoder4` structure but without normalization layers
- **Input:** `images` — same format as fnet
- **Output:** `imap` — `[batch, frames, 384, H/4, W/4]`, float32
- **Purpose:** Produces 384-channel context vectors that initialize the hidden state of
  the GRU update module. These feed the `c1`/`c2` linear layers that condition pose and
  depth refinement.
- **Normalization:** None

## Optimizations applied

These models were generated by applying a sequence of lossless ONNX graph passes using
`onnxoptimizer` (v0.3.13) and `onnx.shape_inference`, followed by ONNX validity checking.
No weights were modified. No quantization was applied. These are basic optimisation passes I used in HeteroAI.

### Results

| Model | Nodes before | Nodes after | Reduction |
|-------|-------------|-------------|-----------|
| fnet  | 105         | 61          | −44 (42%) |
| inet  | 75          | 51          | −24 (32%) |

### Pass-by-pass explanation

#### Operator fusion

**`fuse_add_bias_into_conv`** (largest single contributor)

The ONNX exporter emits convolution bias as a separate `Add` node after each `Conv` node.
This pass folds each `Conv → Add(bias)` pair into a single `Conv` node with the bias
attribute set. Every convolution layer in both encoders is affected — this accounts for
the majority of the node count reduction.

```
Before:  [Input] → Conv (no bias) → Add (bias tensor) → ReLU
After:   [Input] → Conv (bias embedded) → ReLU
```

**`fuse_pad_into_conv`**

Where a `Pad` node immediately precedes a `Conv`, the padding is absorbed into the
Conv's `pads` attribute, eliminating the separate Pad node and the intermediate buffer.

#### Shape/constant specialisation

**`eliminate_shape_gather` / `eliminate_slice_after_shape`**

`BasicEncoder4.forward()` reshapes between `[B*N, C, H, W]` and `[B, N, C, H/4, W/4]`.
The exporter emits this as a chain of `Shape → Gather → Unsqueeze → Concat` nodes that
compute the target shape at runtime. Since H and W are static (known at export time),
these chains are folded into scalar constants at graph-compile time, removing ~10 nodes
per model.

**`extract_constant_to_initializer`**

Remaining `Constant` nodes (inline weight tensors encoded as graph nodes) are promoted
to named initializers. Initializers are loaded once at session creation and never
re-evaluated during inference, reducing per-inference node traversal overhead.

**`eliminate_shape_op`**

`Shape` operations whose output is fully determined by static input shapes are replaced
with constant tensors.

#### Dead node / redundancy elimination

| Pass | Effect |
|------|--------|
| `eliminate_deadend` | Remove nodes whose outputs are never consumed |
| `eliminate_identity` | Remove `Identity` pass-through nodes |
| `eliminate_nop_cast` | Remove type casts that don't change dtype |
| `eliminate_nop_pad` | Remove `Pad` nodes with all-zero padding |
| `eliminate_nop_reshape` | Remove `Reshape` nodes that don't change shape |
| `eliminate_nop_transpose` | Remove `Transpose` nodes with identity permutation |
| `eliminate_nop_with_unit` | Remove multiply-by-1, add-by-0, etc. |
| `eliminate_common_subexpression` | Deduplicate identical subgraphs |
| `eliminate_unused_initializer` | Remove weight tensors not referenced by any node |
| `eliminate_duplicate_initializer` | Deduplicate identical weight tensors |

#### Shape inference (post-pass)

`onnx.shape_inference.infer_shapes()` annotates every intermediate tensor in the graph
with its dtype and shape. ONNX Runtime uses this information to:
- Pre-allocate all intermediate buffers at session load time
- Skip runtime shape computation during inference
- Plan kernel selection and memory layout ahead of time

## Opset and compatibility

- **Opset version:** 17
- **ONNX ops used (fnet):** `Conv`, `InstanceNormalization`, `Relu`, `Add`, `Reshape`, `Constant`
- **ONNX ops used (inet):** `Conv`, `Relu`, `Add`, `Reshape`, `Constant`
- Compatible with any ONNX Runtime version that supports opset 17 (ORT ≥ 1.13)

## How to use

These models accept the same inputs and produce the same outputs as the originals.
Point your inference code at this folder instead of the parent folder.

```python
import onnxruntime as ort
import numpy as np

sess_fnet = ort.InferenceSession("George_ONNX/optimized/fnet.onnx",
                                  providers=["CUDAExecutionProvider", "CPUExecutionProvider"])
sess_inet = ort.InferenceSession("George_ONNX/optimized/inet.onnx",
                                  providers=["CUDAExecutionProvider", "CPUExecutionProvider"])

# images: float32, shape [1, N, 3, H, W], normalized as 2*(x/255)-0.5
fmap = sess_fnet.run(None, {"images": images})[0]  # [1, N, 128, H/4, W/4]
imap = sess_inet.run(None, {"images": images})[0]  # [1, N, 384, H/4, W/4]
```

## Regenerating

To reproduce these files from scratch:

```bash
# 1. Export from dpvo.pth weights
python George_ONNX/export_encoders.py

# 2. Optimize (requires onnxoptimizer and onnx)
python George_ONNX/optimize_encoders.py
# output lands in George_ONNX/optimized/ by default
```

Dependencies: `onnx`, `onnxoptimizer` (installable via pip).

## What was not changed

- **No quantization** — weights remain float32
- **No weight pruning** — all parameters intact
- **No architecture changes** — same layers, same computation
- **Dynamic axes preserved** — batch, frames, height, width remain dynamic
- **InstanceNorm behaviour preserved** — fnet still computes per-batch statistics
  at inference time (correct for visual odometry; not foldable into Conv unlike BatchNorm)
